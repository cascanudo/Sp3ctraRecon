# Security Policy / Politica de Seguridad

[English](#english) | [Espanol](#español)

---

## English

Sp3ctraRecon is designed for education, lab work, and authorized testing.

### Scope

Report issues related to:

- Insecure execution of user inputs
- Writing outside the expected directory
- Accidental data leakage in reports
- Destructive or overly aggressive flags for conservative profiles
- Silent degradation that could hide real errors

### What to Include

If possible, include:

- Affected version
- Command or workflow used
- Example target (if shareable)
- Expected result vs actual result
- Minimal evidence to reproduce the issue

### What We Do With a Report

- Confirm the impact
- Reproduce the issue in a controlled way
- Fix with the smallest safe change possible
- Update documentation or validation if needed

### Responsible Use

Do not use this project to attack systems without explicit authorization. If you detect a weakness in the tool, report it instead of exploiting it.

## Español

Sp3ctraRecon esta disenado para educacion, laboratorio y pruebas autorizadas.

### Alcance

Reporta problemas relacionados con:

- Ejecucion insegura de entradas del usuario
- Escritura fuera del directorio esperado

Incluye, si es posible:

### Que Hacemos Con un Reporte

- Confirmamos el impacto
- Reproducimos el problema de forma controlada
- Corregimos con el menor cambio seguro posible
- Actualizamos documentacion o validacion si hace falta

No uses este proyecto para atacar sistemas sin autorizacion explicita. Si detectas una debilidad en la herramienta, reportala en lugar de explotarla.
